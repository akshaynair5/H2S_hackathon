üìê Technical Overview

TrustMeter leverages a Chrome extension frontend with a Flask backend, integrating AI/ML models, vector databases, and external APIs for efficient misinformation detection.

üåê Frontend Layer - Browser Extension

Lightweight Chrome extension that integrates seamlessly into your browsing experience.

Content Script Injector UI Overlay Manager Context Menu Handler Event Listeners Local Storage Cache API Client

‚öôÔ∏è Backend Layer - Flask API

Python Flask server handling requests, orchestrating AI models, and managing data flow.

REST API Endpoints Request Router Cache Manager (Pinecone) AI Model Orchestrator Domain Scorer (Firestore) Feedback Aggregator

ü§ñ AI/ML Processing Layer

Advanced machine learning models for text analysis, image verification, and credibility scoring.

Gemini AI (Reasoning) Vertex AI (ML Classifier) Google Cloud Vision (Image) Google Fact Check API Google Custom Search Sentence Transformers (Embeddings)

üíæ Data & Storage Layer

Hybrid storage with vector database and document database for caching and feedback.

Pinecone Vector DB (384-dim) Firestore (Domain Scores) Semantic Search (cosine similarity) Anonymous User Tracking TTL Expiration (15 days) Namespace Separation

üîÑ Data Flow

Step 1: Content Detection

Extension monitors webpage for text, images, and links. User triggers verification via context menu or button click.

Step 2: Cache Check

Pinecone semantic search queries for similar content. If found with similarity(cosine and verified fakes search) and 1+ user confirmation, return cached result.

Step 3: AI Analysis

If cache miss, trigger parallel AI processing: Gemini extracts metadata, Vertex AI classifies, Vision analyzes images.

Step 4: Fact Check

Google Fact Check API searches for professional fact-checker ratings. Gemini refines claim for better matching.

Step 5: Evidence Corroboration

Google Custom Search with dynamic keywords finds supporting/contradicting evidence from trusted domains.

Step 6: Ensemble Scoring

Weighted combination: 50% Gemini + 30% Vertex + 20% Fact Check produces final prediction and confidence.

Step 7: Domain Update

Firestore domain scores updated based on verification results. Weighted average by unique user votes.

Step 8: Cache Storage

Final result stored in Pinecone (vector) and Firestore (document) with 15-day TTL.

Step 9: Result Delivery

Findings sent back to extension and displayed in user-friendly overlay with explanation and evidence links.

üõ†Ô∏è Troubleshooting

‚ùå Extension Not Loading?

Ensure Developer mode is enabled in chrome://extensions/ and you've selected the correct /app folder.

‚ö†Ô∏è Verification Not Working?

Check your internet connection and ensure the backend API is running. Try refreshing the webpage and check browser console for errors.

üêå Slow Performance?

Clear your browser cache and disable other extensions temporarily. First-time checks are slower; subsequent checks use Pinecone cache.

üîë API Key Issues?

Verify all API keys are correctly set in your .env file: GEMINI_API_KEY, PINECONE_API, GOOGLE_API_KEY, etc.

üíæ Cache Not Working?

Check Pinecone index status and ensure Firestore rules allow read/write. Run cleanup_expired() if cache is stale.

üìä Inaccurate Results?

Provide feedback through the extension to improve accuracy. System learns from unique user votes and updates domain scores.
